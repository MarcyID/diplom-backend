# Краткая справка по технологиям для диплома

## 📌 Основное для включения в диплом

### 1. Backend технологии

| Технология | Назначение | Обоснование |
|------------|------------|-------------|
| **Go (Golang) 1.21+** | Язык программирования | Высокая производительность, статическая типизация, встроенная поддержка HTTP |
| **Gin Web Framework** | Веб-фреймворк | Производительность до 40× быстрее net/http, middleware, валидация JSON |
| **PostgreSQL 15** | База данных | ACID-транзакции, внешние ключи, индексы, триггеры |
| **Redis / In-Memory Cache** | Кеширование | Экономия лимитов API на 98%, TTL 24 часа |
| **JWT (golang-jwt/jwt/v5)** | Аутентификация | Stateless аутентификация, Access Token 15 мин, Refresh Token 7 дней |
| **bcrypt** | Хеширование паролей | Необратимое хеширование, стоимость 10 |

---

### 2. Архитектура приложения

**Слоёная архитектура:**
```
HTTP Layer (Gin) → Service Layer (бизнес-логика) → Repository Layer (БД) → Data Layer (PostgreSQL/Redis)
```

**Принципы:**
- Разделение ответственности (Separation of Concerns)
- Dependency Injection
- 12-Factor App (конфигурация через переменные окружения)

---

### 3. Развёртывание (Deployment)

#### Frontend — GitHub Pages

**GitHub Pages** — бесплатный хостинг статических сайтов.

**Процесс развёртывания:**
1. Сборка фронтенда: `npm run build`
2. GitHub Actions автоматически публикует в ветку `gh-pages`
3. Доступ по URL: `https://username.github.io/repo-name`

**Преимущества:**
- Бесплатно
- Интеграция с GitHub
- HTTPS из коробки
- Автоматический деплой при push в main

#### Backend — Render (Cloud PaaS)

**Render** — платформа для развёртывания веб-приложений.

**Процесс развёртывания:**
1. Подключение GitHub репозитория
2. Автоматическая сборка: `go build -o server ./cmd/server`
3. Запуск: `./server`
4. PostgreSQL добавляется как аддон

**Преимущества:**
- Бесплатный тариф
- Автоматический деплой из GitHub
- Встроенная поддержка PostgreSQL
- HTTPS автоматически

---

### 4. Безопасность

| Мера защиты | Реализация |
|-------------|------------|
| Хеширование паролей | bcrypt (стоимость 10) |
| JWT токены | HMAC-SHA256 подпись |
| Refresh Token | Хеширование SHA-256 перед сохранением в БД |
| CORS | Настройка разрешённых origins |
| Валидация входных данных | Gin validator + проверки в сервисах |

---

### 5. Производительность

**Метрики:**
- Время ответа из кеша: **1-5 мс**
- Время ответа от Kinopoisk API: **100-500 мс**
- RPS (Requests Per Second): **до 1000** (без БД)

**Оптимизации:**
- Connection pooling (pgxpool)
- Keep-Alive для HTTP соединений
- In-Memory кеш для статических данных
- Индексы в БД (email, username)

### 6. Структура проекта

```
diplom-backend/
├── cmd/server/main.go          # Точка входа
├── internal/
│   ├── api/                    # HTTP слой
│   │   ├── router.go
│   │   ├── middleware/auth.go  # JWT middleware
│   │   └── handlers/           # Обработчики запросов
│   ├── service/                # Бизнес-логика
│   │   ├── kinopoisk_client.go # API клиент с кешем
│   │   └── auth_service.go     # Аутентификация
│   ├── repository/             # Работа с БД
│   │   ├── user_repository.go
│   │   └── session_repository.go
│   ├── cache/                  # Кеширование
│   │   ├── memory_cache.go
│   │   └── redis_cache.go
│   └── model/                  # Модели данных
├── api/                        # Документация
│   ├── openapi/
│   └── postman/
└── .env                        # Конфигурация
```

### 7. Конфигурация (переменные окружения)

```env
# API
KINOPOISK_API_KEY=ваш_ключ
PORT=5454

# База данных
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=diplom_db

# Безопасность
JWT_SECRET=секретный_ключ

# CORS
ALLOWED_ORIGINS=https://username.github.io

# Кеширование
CACHE_TYPE=memory
CACHE_TTL_HOURS=24
```

### Для введения:
> «В рамках дипломного проекта разработан backend для системы поиска и рекомендаций фильмов на языке программирования Go с использованием веб-фреймворка Gin. Для хранения данных пользователей применена СУБД PostgreSQL, для кеширования запросов к внешнему API — Redis/In-Memory Cache. ### Для главы «Анализ технологий»:
> «Выбор языка Go обусловлен его высокой производительностью благодаря компиляции в машинный код, встроенной поддержкой многопоточности (горутины) и статической типизацией, предотвращающей ошибки на этапе компиляции. > «Приложение построено по слоёной архитектуре: HTTP слой (Gin handlers) → Service слой (бизнес-логика) → Repository слой (работа с БД) → Data слой (PostgreSQL/Redis). Такое разделение обеспечивает соблюдение принципа единственной ответственности и упрощает тестирование.»

> «Frontend размещён на GitHub Pages — бесплатном хостинге статических сайтов с автоматическим деплоем через GitHub Actions. Backend развёрнут на платформе Render, предоставляющей бесплатный тариф с автоматическим масштабированием и встроенной поддержкой PostgreSQL. CORS настроен для взаимодействия между доменами github.io и onrender.com.»
